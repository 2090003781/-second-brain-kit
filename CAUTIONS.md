# Cautions / 注意事项

> **For AI agents (MANDATORY):** When you first encounter this project, you MUST read this file and present EACH item to the user as a question or notification. Do not skip, do not assume defaults. Each section has a structured format — follow it exactly. Only proceed to setup or execution after all items have been addressed.
>
> **For humans:** 这份文件列出了搭建和使用本系统时需要你知晓和配合的所有事项。AI agent 会逐条向你提问，你回答后它再继续。

---

## 1. 如果你已经在用 Obsidian

**风险等级：** 🔴 高 — 可能导致已有文件被覆盖

**需要你配合：** 确认 vault 路径 + 选择测试还是直用

**Agent 应该这样说：**

> "我看到这个系统会把一套模板文件（规则库、错误库、待办清单等）复制到你的 Obsidian vault 里。如果你已经在用 Obsidian，你 vault 里可能已经有同名文件了。
>
> 你目前是用 Obsidian 的，还是第一次用？
> - **我在用，有一个正在用的 vault** — 建议先新建一个空文件夹做测试：`mkdir D:\测试-第二大脑`，确认系统正常后再迁移到正式 vault
> - **第一次用** — 那直接设置就行，vault 就是为这个系统建的
> - **我在用，但清楚风险，直接往现有的 vault 装** — 也可以，但建议先手动备份一次"

**Agent 收到回答后：**
根据回答决定使用哪个路径。如果是已有 vault 且选了直装，提醒一次后尊重选择。

---

## 2. 系统需要定时任务来维护自己

**风险等级：** 🟡 中 — 不设也能用，但知识不会自动增长

**需要你配合：** 确认是否创建定时任务

**Agent 应该这样说：**

> "这个系统有两个维护脚本，需要定时运行才能持续生效：
> - **每日提炼**（凌晨 4:00）— 把当天的会话日志提炼成错误模式和经验，更新知识库
> - **每周自检**（周日 5:00）— 检查规则矛盾、断链、知识陈旧
>
> 不设置的话系统仍然能用，但知识库不会自动增长，错误模式也不会更新。
>
> 要现在设置定时任务吗？
> - **帮我设置** — Windows Task Scheduler 或 cron 我来指导
> - **稍后我自己设**
> - **不用，我手动跑**

**Agent 收到回答后：**
如果选"帮我设置"，引导用户打开 Task Scheduler（Windows）或 crontab（macOS/Linux）。如果选稍后或不用，记录这个决定。

---

## 3. 守护进程需要在后台运行

**风险等级：** 🟡 中 — 不运行则会话日志写不进 vault

**需要你配合：** 确认是否启动 + 是否开机自启

**Agent 应该这样说：**

> "这个系统有两个后台程序（守护进程），负责把每次 AI 会话的日志实时写入你的 Obsidian vault：
> - `obsidian_writer`（端口 49520）
> - `supervisor`（端口 49522）
>
> 它们需要一直运行才能工作。你希望：
> - **现在启动，并设置开机自启**
> - **现在启动，下次开机我手动开**
> - **暂不启动，需要时再说**

**Agent 收到回答后：**
如果选择启动，运行对应的启动命令。如果需要开机自启，引导添加 Windows 开机启动项或 systemd service。

## 4. 知识库初始是空的

**风险等级：** 🟢 低 — 正常现象

**需要你配合：** 知晓即可

**Agent 应该这样说：**

> "知识库目前只包含示例文件，没有实际内容。这是正常的——知识会随着你使用系统慢慢积累：
> - 每次 AI 会话自动写入 `话题/` 目录
> - 每日提炼从中提取错误模式和经验
> - 你也可以手动往 `知识库/` 添加笔记
>
> 大概用 2-3 周后，你就会有一个可用的个人知识库了。"

**Agent 收到回答后：**
解释完即可，无需操作。

## 5. 监督程序初期可能误报

**风险等级：** 🟢 低 — 不影响使用

**需要你配合：** 知晓即可

**Agent 应该这样说：**

> "监督程序是**关键词匹配**的，不是 AI 语义判断。初期可能会出现：
> - 你路径里有中文就报 GBK 编码警告——如果你的 vault 路径本身就是中文，这是正常现象，不影响使用
> - 写配置文件时提示要先备份——这个建议是有用的
>
> 误报率会随着每日提炼的进行逐渐降低。如果某个警告明显是误报，忽略就好。"

**Agent 收到回答后：**
解释完即可，无需操作。

## 6. vault 路径硬编码在多处

**风险等级：** 🟡 中 — 迁移时需要手动改

**需要你配合：** 知晓 + 迁移时记得

**Agent 应该这样说：**

> "你的 Obsidian vault 路径目前硬编码在多个配置文件中。如果你以后换电脑或换路径，需要手动更新这些地方。建议到时候找我或记录一下有哪些文件要改。"

**Agent 收到回答后：**
记录这个注意事项。如果用户要求，可以现在就把路径集中到一个配置文件里。

## 7. 缓存优化可节省 API 费用

**风险等级：** 🟢 低 — 优化项，非必需

**需要你配合：** 知晓 + 选择是否应用

**Agent 应该这样说：**

> "这套系统可以优化 AI 的 token 缓存命中率，节省 API 调用费用。做法是把稳定的内容（工具定义、核心规则）放在上下文前面，把频繁变化的内容（知识索引）改成按需读取。
>
> 这个优化我已经准备好了，要现在应用吗？
> - **应用**
> - **先不用，以后再说**

**Agent 收到回答后：**
如果选择应用，调整 AGENTS.md 和加载顺序。如果不应用，记录决定。
