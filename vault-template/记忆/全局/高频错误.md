# High-Frequency Errors — Global Domain

Common mistakes that recur across domains. Each entry documents the phenomenon, frequency of occurrence, and proven solution.

---

## #1 — GBK / Unicode encoding conflict

- **次数：** 12
- **现象：** Commands or tool arguments containing CJK characters (e.g., Chinese file paths) trigger `UnicodeDecodeError` or garbled output on Windows when the active code page is GBK.
- **解决：** Pass CJK strings via environment variables or config values. For Go files use `encoding='gbk'`; for Python scripts always specify `encoding='utf-8'`. Avoid embedding CJK literals inline in shell commands.

---

## #2 — Modifying config files without backup

- **次数：** 8
- **现象：** `write_file` is invoked on a `.toml`, `.json`, or `.yaml` config file before a backup copy is created.
- **解决：** Always run `Copy-Item <path> <path>.bak` (or the Unix equivalent) before overwriting a configuration file.

---

## #3 — Absolute user path hardcoded

- **次数：** 6
- **现象：** Scripts or configurations reference an absolute path that only exists on one machine (e.g., `D:\个人数据\辞玖`, `C:\Users\20900\.reasonix`).
- **解决：** Use config variables (`config.toml`), environment variables, or relative paths. The project provides `config.py` for centralised path resolution.

---

## #4 — Asking to delete without explaining impact

- **次数：** 5
- **现象：** The AI attempts to delete files or code sections without first explaining what will be removed and why.
- **解决：** Before any delete operation, output a clear statement: "I will delete X because Y. The affected areas are Z."

---

## #5 — Running foreground process that hangs

- **次数：** 4
- **现象：** A long-running process (e.g., bot, file watcher, dev server) is started in the foreground, causing the shell tool to block until timeout.
- **解决：** Use `Start-Process -NoNewWindow` (Windows) or `nohup` / `&` (macOS/Linux) for services that run longer than a few seconds.
